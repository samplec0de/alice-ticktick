# Дизайн: FR-15 — Фильтрация задач по приоритету

## Контекст

Фаза 4 PRD. FR-13 (повторяющиеся задачи), FR-14 (напоминания), FR-21 (длительность) уже реализованы.
FR-12 (теги) отложен — требует V2 API.

Остаётся FR-15 — комбинированная фильтрация: дата + приоритет.

## Подход

Расширяем существующий интент `list_tasks` слотом `priority`.

## Изменения

### 1. Интент `list_tasks` (Яндекс Диалоги)

Добавить слот `priority` (type: YANDEX.STRING) в грамматику.

Примеры фраз:
- "покажи срочные задачи на завтра"
- "какие важные задачи на эту неделю"
- "что с высоким приоритетом на сегодня"

### 2. `intents.py` — `ListTasksSlots`

Добавить поле `priority: str | None = None`. Обновить `extract_list_tasks_slots()`.

### 3. `handlers.py` — `handle_list_tasks()`

После получения задач за дату — фильтровать по `parse_priority(slots.priority)` если слот заполнен.
Обновить текст ответа с указанием приоритета.

### 4. `responses.py`

Новые шаблоны: `TASKS_FOR_DATE_WITH_PRIORITY`, `NO_TASKS_FOR_DATE_WITH_PRIORITY`.

### 5. Тесты

- `handle_list_tasks` с заполненным priority-слотом
- Фильтрация: задачи с разными приоритетами, пустой результат после фильтра
- `extract_list_tasks_slots` с priority
