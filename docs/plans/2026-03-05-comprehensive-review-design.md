# Дизайн: Комплексное ревью интеграции Alice-TickTick

**Дата:** 2026-03-05
**Цель:** Найти и исправить баги, UX-проблемы и code smell-ы через коллегиальный многоагентный процесс.

## Выход

Готовые PR-ы с фиксами. Сначала UX → логика интентов → код.

## Команда агентов

| Агент | Роль |
|-------|------|
| `team-lead` | Координирует, собирает отчёты, проводит дискуссии, создаёт PR-ы |
| `ux-reviewer` | Анализирует тексты ответов, TTS, длину, сценарии диалога |
| `logic-reviewer` | Анализирует disambiguation, edge cases, NLU-срабатывания |
| `code-reviewer` | Анализирует структуру кода, дублирование, handlers.py |
| `critic` | После каждого отчёта атакует слабые места, ищет пропущенные случаи |

## Процесс

1. **Параллельный анализ** — ux-reviewer, logic-reviewer, code-reviewer работают одновременно, каждый по своей области
2. **Критика** — critic получает каждый отчёт и атакует его: "а что если...", "ты упустил..."
3. **Защита** — reviewer отвечает на критику, уточняет или отзывает findings
4. **Сведение** — team-lead собирает финальный список issues
5. **Дискуссия по спорным** — для неоднозначных issues: reviewer защищает, critic атакует, team-lead принимает решение
6. **Реализация** — team-lead (или implementer) создаёт PR на каждый принятый issue

## Области анализа

### UX (`ux-reviewer`)
- Тексты в `responses.py`: естественность, последовательность, стиль
- TTS: читаемость вслух, знаки препинания, скобки
- Лимит 1024 символа: обрезаются ли длинные списки корректно
- Сценарии ошибок: что говорится при API error, not found, missing slot
- Брифинги: утренний и вечерний, формулировки
- Welcome/goodbye: первый запуск vs возвращение

### Логика интентов (`logic-reviewer`)
- `router.py`: порядок интентов, потенциальные конфликты
- Disambiguation `add_subtask` vs `create_task`
- `_try_parse_checklist_command`: покрытие фраз, граничные случаи
- FSM для delete: поведение при cold start, таймаут состояния
- `overdue_tasks` + `list_tasks` + `project_tasks`: корректность фильтров дат
- Edge cases: пустое имя задачи, спецсимволы, очень длинные фразы

### Код (`code-reviewer`)
- `handlers.py` 2212 строк: можно ли разбить на модули
- Дублирование логики фильтрации задач (`_apply_task_filters`)
- Смешение NLP-логики и ответов в handlers
- Обработка ошибок: где `except Exception` без логирования
- Типизация: Any в сигнатурах, непокрытые cases
