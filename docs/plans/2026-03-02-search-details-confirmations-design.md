# Дизайн: Детали поиска и подтверждения действий

## Цель

1. При поиске задачи — показывать описание и чеклист (со статусами) лучшего совпадения
2. На каждое успешное действие — подтверждение с описанием что именно было сделано

## 1. Поиск задачи (`search_task`)

### Текущее поведение

Возвращает пронумерованный список названий задач (до 5) с датой/приоритетом.

### Новое поведение

Лучшее совпадение показывается с полными деталями, остальные — только названиями.

**Формат ответа:**
```
Лучшее совпадение — «Купить продукты» (завтра, средний приоритет):
Описание: Зайти в Перекрёсток, взять всё по списку
Чеклист:
1. [x] Молоко
2. [ ] Хлеб
3. [ ] Яблоки

Также найдено:
2. Отчёт по продуктивности (сегодня)
3. Продуктовый бюджет
```

Если описания/чеклиста нет — соответствующие секции пропускаются.
Если найдена ровно 1 задача — секция «Также найдено» отсутствует.

### Логика сборки с бюджетом символов

Приоритет заполнения (в порядке убывания важности):
1. Заголовок лучшего совпадения с датой/приоритетом
2. Описание (`content`) — показать целиком, обрезать если не влезает
3. Чеклист (`items`) — показать сколько влезет, добавить «и ещё N пунктов» если обрезано
4. Остальные совпадения списком

Бюджет = 1024 символа (лимит Алисы).

### Формат чеклиста

```
1. [x] Выполненный пункт
2. [ ] Невыполненный пункт
```

## 2. Улучшенные подтверждения действий

### `edit_task`

**Было:** `Задача "Отчёт" обновлена.`
**Стало:** `Задача "Отчёт" обновлена: дата изменена на завтра, приоритет — высокий.`

Перечислять все изменённые поля через запятую. Примеры полей:
- дата изменена на {date}
- приоритет — {priority}
- перемещена в проект "{project}"

Частные случаи (повторение, напоминание) уже обработаны отдельными шаблонами.

### `complete_task`

**Было:** `Задача "Купить молоко" отмечена выполненной.`
**Стало:** `Задача "Купить молоко" (сегодня, низкий приоритет) отмечена выполненной.`

Добавить дату и приоритет в скобках если есть.

### `delete_task`

**Было:** `Задача "Старый отчёт" удалена.`
**Стало:** `Задача "Старый отчёт" (15 мая) удалена.`

Добавить дату в скобках если есть.

### `create_task`

**Было:** `Готово! Задача "Отчёт" создана на завтра.`
**Стало:** `Готово! Задача "Отчёт" создана на завтра, приоритет — высокий.`

Добавить приоритет если указан (кроме NONE). Шаблоны с повторением/напоминанием/проектом — оставить как есть, но тоже добавить приоритет.

### `check_item`

**Было:** `Пункт "Молоко" отмечен выполненным.`
**Стало:** `Пункт "Молоко" в задаче "Купить продукты" отмечен выполненным.`

Добавить название задачи.

### Без изменений

- `add_subtask` — уже показывает подзадачу и родителя
- `add_checklist_item` — уже показывает пункт, задачу и количество
- `delete_checklist_item` — уже показывает пункт и задачу
- `list_tasks`, `list_subtasks`, `show_checklist`, `overdue_tasks` — просмотр, не действие

## Затронутые файлы

- `alice_ticktick/dialogs/handlers.py` — логика search_task, complete_task, edit_task, delete_task, create_task, check_item
- `alice_ticktick/dialogs/responses.py` — шаблоны ответов
- `tests/` — тесты на все изменённые обработчики

## Тестирование

- Юнит-тесты на каждый изменённый обработчик
- Тесты на бюджет символов (задача с длинным описанием + чеклистом)
- Тесты на граничные случаи: пустое описание, пустой чеклист, одно совпадение
