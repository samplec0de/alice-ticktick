# Результаты комплексного ревью Alice-TickTick

**Дата:** 2026-03-05

## HIGH (7 issues) — исправить в первую очередь

### L-10/C-6: Глобальный кэш не привязан к пользователю
**Серьёзность:** high (privacy issue)
**Проблема:** Кэш данных TickTick не привязан к access_token — на warm Lambda instance один пользователь может видеть данные другого.
**Фикс:** Привязать кэш к access_token пользователя.

### L-1: complete_task без подтверждения при низком fuzzy score
**Серьёзность:** high
**Проблема:** complete_task выполняется без подтверждения при score < 60. Деструктивная операция.
**Фикс:** Запрашивать подтверждение при score < 85-90.

### L-2: parse_yandex_datetime без `now` в list_tasks и project_tasks
**Серьёзность:** high
**Проблема:** parse_yandex_datetime вызывается без `now` — неправильные даты для восточных часовых поясов.
**Фикс:** Передавать `now=datetime.datetime.now(tz=user_tz)`.

### UX-4/UX-14: Двойное двоеточие + тавтология в OVERDUE_TASKS_HEADER
**Серьёзность:** high
**Проблема:** "Просроченных задач: {count}:" — двойное двоеточие и тавтология "задач N задач".
**Фикс:** "Просроченных — {count}:"

### UX-5: Аналогичная проблема в OVERDUE_WITH_PRIORITY
**Серьёзность:** high
**Фикс:** Исправить вместе с UX-4.

### UX-6: Некорректное склонение приоритета после "с"
**Серьёзность:** high
**Проблема:** "с высокий приоритет" вместо "с высоким приоритетом" (творительный падеж).
**Фикс:** Добавить `_format_priority_instrumental()`.

### C-2: 10 копипаст-блоков fuzzy match по 8 строк каждый
**Серьёзность:** high
**Проблема:** Один и тот же паттерн fuzzy match + error handling дублируется 10+ раз.
**Фикс:** Хелпер `_find_active_task() -> TaskMatch | None`.

## MEDIUM (15 issues)

### UX-1: Непоследовательный стиль "Готово!" vs "Добавила!"
**Проблема:** Разные глагольные формы при создании задач. Ж.р. глаголы смешаны с безличными.
**Фикс:** Унифицировать на "Готово!" для всех TASK_CREATED_*.

### UX-2: "Отменила удаление" → безличная форма
**Фикс:** "Удаление отменено."

### UX-3: "Добавила «item»" → пассивный залог
**Фикс:** "Пункт «item» добавлен..."

### UX-13 (найден critic): Глаголы ж.р. в error-ответах
**Проблема:** "Не поняла команду", "Не поняла, что изменить" — тот же паттерн что UX-1/2/3.
**Фикс:** Унифицировать на безличную форму.

### UX-7: Непоследовательные кавычки "..." vs «...»
**Фикс:** Унифицировать на «ёлочки» везде.

### UX-8: handle_list_projects не вызывает `_truncate_response`
**Фикс:** Одна строка — добавить вызов.

### UX-9: _truncate_response обрезает посреди слова
**Фикс:** Обрезать по последнему `\n` перед лимитом.

### UX-10: Брифинги не показывают "и ещё N задач"
**Проблема:** Показывают 5 задач но указывают полное количество без пояснения что остальные скрыты.
**Фикс:** Добавить "...и ещё N задач." в конец списка.

### L-7: Стоп-слова не проверяются в complete/edit/delete/search
**Фикс:** Добавить проверку `_TASK_NAME_STOPWORDS` в эти handlers.

### L-8: Timezone fallback на UTC без warning
**Фикс:** `logger.warning("No timezone in request, falling back to UTC")`.

### L-11 (найден critic): edit_task тоже использует fuzzy без подтверждения
**Фикс:** Аналогично L-1 — запрашивать подтверждение при низком score.

### C-1: handlers.py 2212 строк — God Module
**Фикс:** Разбить на 4-5 тематических модулей. Низкий приоритет, большой объём работы.

### C-3: Фильтрация в briefing дублирует _apply_task_filters
**Фикс:** Переиспользовать `_apply_task_filters`.

### C-5: Широкий except Exception без разграничения типов
**Фикс:** Обрабатывать `TickTickUnauthorizedError` отдельно (→ Account Linking ответ).

### C-7: handle_create_task 300 строк
**Фикс:** Вынести response building в отдельный helper. Низкий приоритет.
