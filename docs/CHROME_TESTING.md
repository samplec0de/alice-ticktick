# Тестирование навыка через Chrome (browser automation)

Инструкция для тестирования навыка «ТикТик» через консоль Яндекс.Диалогов с помощью Chrome browser automation (mcp__claude-in-chrome).

## 1. Подготовка сессии

### 1.1. Начало работы

1. Вызвать `mcp__claude-in-chrome__tabs_context_mcp` — получить контекст текущих вкладок.
2. Создать новую вкладку: `mcp__claude-in-chrome__tabs_create_mcp`.
3. Перейти на страницу тестирования:
   ```
   https://dialogs.yandex.ru/developer/skills/d3f073db-dece-42b8-9447-87511df30c83/draft/test
   ```
4. Дождаться загрузки страницы (может потребоваться авторизация в Яндексе).

### 1.2. Проверка загрузки

- На странице должен быть виден чат-интерфейс с полем ввода.
- Если страница требует авторизации — сообщить пользователю.

## 2. Соглашения об именовании тестовых задач

При создании задач в ходе тестирования **всегда включай слово «тестирование» в название задачи**, например:

- `создай задачу тестирование купить хлеб`
- `добавь задачу тестирование позвонить маме на завтра`
- `создай задачу тестирование встреча завтра в 10 на 2 часа`

Это позволяет легко найти и удалить все тестовые задачи командой `найди задачу тестирование` после завершения тестовой сессии.

---

## 3. Как отправлять команды

### 2.1. Отправка сообщения

1. Найти поле ввода текста на странице тестирования.
2. Использовать `mcp__claude-in-chrome__form_input` для ввода текста команды.
3. Нажать Enter или кнопку отправки.

### 2.2. Чтение ответа

1. После отправки дождаться ответа (1-3 секунды).
2. Использовать `mcp__claude-in-chrome__get_page_text` или `mcp__claude-in-chrome__read_page` для чтения ответа навыка.
3. Проверить:
   - Текст ответа соответствует ожидаемому шаблону.
   - Интент распознан верно (если видно в JSON).
   - Слоты извлечены корректно.

### 2.3. Начало новой сессии

Для тестирования приветствия или сброса состояния:
- Обновить страницу (`mcp__claude-in-chrome__navigate` на тот же URL).
- Это создаёт новую сессию навыка.

## 4. Тестовые сценарии

Для каждого сценария указаны: команда, ожидаемый ответ, и на что обращать внимание.

Результат каждого теста:
- **OK** — ответ соответствует ожиданию
- **WARN** — работает с замечаниями (описать)
- **FAIL** — не работает (описать причину)

## 5. Известные баги (проверка регрессии)

Эти сценарии документируют известные проблемы. При тестировании проверять, исправлены ли они.

| # | Команда | Ожидаемый интент | Известная проблема |
|---|---------|------------------|--------------------|
| 1 | `перенеси задачу сменить полотенца на завтра` | edit_task (date=завтра) | `.+` в грамматике поглощает «на завтра» |

## 6. Порядок полного прохода

Рекомендуемый порядок тестирования для минимизации побочных эффектов:

1. **Приветствие** (4.1) — обновить страницу, проверить приветствие
2. **Создание задач** (4.2) — создать несколько тестовых задач
3. **Просмотр задач** (4.3) — убедиться, что созданные задачи видны
4. **Фильтрация** (4.4) — проверить фильтры
5. **Просроченные** (4.5) — проверить просроченные
6. **Поиск** (4.7) — найти созданные задачи
7. **Редактирование** (4.8) — изменить даты, приоритеты, повторения, напоминания
8. **Подзадачи** (4.12) — добавить и посмотреть подзадачи
9. **Чеклисты** (4.13) — добавить, посмотреть, отметить, удалить пункты
10. **Проекты** (4.14) — посмотреть, создать, переместить
11. **Напоминания** (4.10, 4.11) — создать повторяющиеся, добавить напоминания
12. **Встречи** (4.2, встречи) — создать задачи с длительностью
13. **Брифинги** (4.15) — утренний/вечерний
14. **Завершение** (4.6) — отметить задачи выполненными
15. **Удаление** (4.9) — удалить тестовые задачи (с подтверждением)
16. **Служебные** (4.16) — помощь, прощание
17. **Fallback** (4.17) — неизвестные команды
18. **Edge cases** (4.18) — длинные названия, числа, пустые списки
19. **Регрессия** (5) — проверить известные баги
