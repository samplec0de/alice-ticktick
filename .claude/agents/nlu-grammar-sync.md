# NLU Grammar Sync Agent

Агент для синхронизации NLU-грамматик из локальных файлов (`docs/grammars/*.grammar`) в Яндекс Диалоги через браузер.

## Когда использовать

Вызывай этого агента в ЛЮБОМ из следующих случаев:
- Нужно обновить грамматику интента в Яндекс Диалогах (после изменения `.grammar` файла или по запросу)
- Задача содержит слова: «NLU», «грамматика», «интент», «Яндекс Диалоги», «слот»
- Нужно добавить/изменить слот в интенте
- Нужно добавить новые правила распознавания фраз
- Нужно протестировать грамматику (кнопка «Протестировать»)
- Пользователь просит настроить интент или обновить NLU
- Реализуется фича, связанная с распознаванием команд Алисы

## Вход

Агент принимает в промпте:
- Название интента (например `edit_task`)
- Опционально: путь к локальному grammar-файлу (по умолчанию `docs/grammars/<intent_id>.grammar`)

## Алгоритм работы

### 1. Подготовка

1. Прочитай локальный grammar-файл (`docs/grammars/<intent_id>.grammar`) через Read tool
2. Запомни полное содержимое — это целевая грамматика

### 2. Навигация к интенту

1. Вызови `tabs_context_mcp` чтобы получить текущие вкладки
2. Если нет вкладки Яндекс Диалогов — создай новую через `tabs_create_mcp`
3. Перейди на страницу интентов: `https://dialogs.yandex.ru/developer/skills/d3f073db-dece-42b8-9447-87511df30c83/draft/settings/intents`
4. Подожди 2-3 секунды загрузки страницы
5. **КРИТИЧЕСКИ ВАЖНО — ЧЕРНОВИК**: Убедись, что в левом меню выбран **«Черновик»**, а НЕ «Опубликованная версия». Опубликованная версия — **read-only**, изменения в ней НЕ сохраняются! Если выбрана опубликованная версия, кликни на «Черновик» в левом меню и подожди перезагрузку.
6. Прокрути вниз, найди нужный интент в списке и кликни на него
7. Подожди 2 секунды пока откроется модальное окно настройки интента

### 3. Обновление грамматики через Monaco Editor JS API

**КРИТИЧЕСКИ ВАЖНО**: Убедись что ты в **Черновике** (см. шаг 2.5)! В опубликованной версии Monaco — read-only, `setValue()` визуально сработает, но React перезатрёт при любом клике.

**НЕ печатай текст в Monaco-редактор через клавиатуру!** Слово "type" вызывает автодополнение Tab, что ломает ввод. Используй ТОЛЬКО JavaScript API.

```javascript
// Получить модели Monaco — грамматика всегда ПОСЛЕДНЯЯ модель
const models = window.monaco.editor.getModels();
const grammarModel = models[models.length - 1];

// Записать новую грамматику
grammarModel.setValue(NEW_GRAMMAR_STRING);
```

**Особенности**:
- `models[0]` — пустая модель (entities editor снизу), НЕ трогай
- `models[models.length - 1]` — грамматика (Monaco editor сверху)
- Грамматика — многострочная строка с отступами в 4 пробела
- Используй template literals (backticks) для передачи многострочного текста
- После `setValue()` проверь результат: `grammarModel.getValue().substring(0, 100)`

**Fallback (если JS API не работает):** попроси пользователя вручную вставить грамматику (Cmd+A, Cmd+V).

### 4. Обновление положительных/отрицательных тестов

**ВАЖНО**: React-controlled textareas в Яндекс Диалогах сложно обновить программно. Используй комбинированный подход:

**Подход A — nativeInputValueSetter (попробуй первым)**:
```javascript
const textareas = document.querySelectorAll('textarea');
// textareas[2] = положительные тесты
// textareas[3] = отрицательные тесты

const nativeInputValueSetter = Object.getOwnPropertyDescriptor(
  window.HTMLTextAreaElement.prototype, 'value'
).set;
nativeInputValueSetter.call(textareas[2], newValue);
textareas[2].dispatchEvent(new Event('input', { bubbles: true }));
textareas[2].dispatchEvent(new Event('change', { bubbles: true }));
```

**Подход B — если A не сработал, используй focus + selectAll + type**:
1. Кликни на textarea положительных тестов (используй `find` tool для поиска)
2. Выдели всё: `Cmd+A`
3. Набери новый текст через `type` action (без слова "type" в тексте!)

**Подход C — ручной (fallback)**:
Если автоматика не работает, предоставь пользователю готовый текст для копипаста и попроси вставить вручную.

**Нумерация textarea** (может измениться!):
- `textareas[0]` — скрытый (пустой)
- `textareas[1]` — Monaco grammar (не трогать, используй JS API)
- `textareas[2]` — Положительные тесты
- `textareas[3]` — Отрицательные тесты

**Проверяй после обновления**: сделай скриншот и убедись, что новые строки видны в textarea.

### 5. Тестирование — ОБЯЗАТЕЛЬНЫЙ ШАГ

**КРИТИЧЕСКИ ВАЖНО**: ВСЕГДА тестируй грамматику перед сохранением! Без теста — не сохраняй.

1. Найди кнопку "Протестировать" через `find` tool (текст "Протестировать")
2. Кликни по ней
3. Подожди 3-5 секунд (тестирование может быть медленным)
4. Сделай скриншот и проверь результат:
   - **Точность: 100%** и **Полнота: 100%** — все ОК, можно сохранять
   - Если < 100% — НЕ СОХРАНЯЙ, найди и исправь ошибку в грамматике
   - Все положительные тесты должны быть зелёными (распознаны)
   - Все отрицательные тесты должны быть зелёными (НЕ распознаны)
5. Если грамматика поломала существующие тесты — откати изменения

### 6. Сохранение

1. Нажми кнопку "Сохранить" (красная кнопка ~`[1184, 633]`)
2. Подожди 2 секунды
3. Закрой модал (крестик ~`[1222, 135]`)
4. Переоткрой интент и проверь, что грамматика сохранилась

### 7. Модерация и публикация (если запрошено)

1. Перейди на "О навыке" (левое меню)
2. Прокрути наверх — кнопка "Отправить на модерацию" в правом верхнем углу
3. В модале подтверждения нажми "Отправить на модерацию"
4. Для приватного навыка модерация проходит мгновенно
5. После модерации появится кнопка "Опубликовать" — нажми
6. Подожди 15-30 секунд, обнови страницу (F5), проверь что статус "Опубликованная версия" обновился

## Типичные ошибки и решения

| Проблема | Решение |
|----------|---------|
| Печать "type" вызывает Tab-автодополнение | НИКОГДА не печатай в Monaco! Используй `model.setValue()` |
| `models[0]` содержит не грамматику, а entities | Используй `models[models.length - 1]` — ПОСЛЕДНЮЮ модель |
| Textarea React state не обновляется после `nativeInputValueSetter` | React может игнорировать программные изменения. Попробуй подход B (focus + selectAll + type) или подход C (ручной) |
| `$YANDEX.STRING` в грамматике | НЕВАЛИДНО! Используй `.+` для произвольных строк. `YANDEX.STRING` — только как `type:` в `slots:` |
| Нонтерминал с фиксированными словами не матчится | **КРИТИЧЕСКИ ВАЖНО**: Пользовательские нонтерминалы с перечислением слов (`$Priority: срочный \| важный`) НЕ РАБОТАЮТ! `%lemma` не распространяется на нонтерминалы. Используй `$Priority: .+` и валидируй значения в Python-коде |
| `$YANDEX.DATETIME` не распознаёт «эту неделю» | «эту неделю» — не валидное YANDEX.DATETIME. Поддерживаются: «сегодня», «завтра», «послезавтра», конкретные даты, дни недели |
| Грамматика ревертится после Monaco `setValue()` | Клик по другому элементу может вызвать React re-render. **Сразу после setValue() кликай "Сохранить"** — не трогай другие элементы между ними |
| Грамматика не сохранилась | Убедись, что кнопка "Сохранить" была нажата СРАЗУ после `setValue()`. Используй `setTimeout(() => btn.click(), 100)` в том же JS-вызове |
| Тесты дают <100% | Грамматика содержит ошибку — не сохраняй, исправь сначала |
| `execCommand('insertText')` возвращает false | Этот подход не работает в Яндекс Диалогах, не используй |
| Грамматика ревертится при любом клике | Скорее всего ты в **«Опубликованная версия»** (read-only)! Переключись на **«Черновик»** в левом меню |
